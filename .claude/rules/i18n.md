# i18n Rules

All user-facing text in the frontend MUST use the `t()` function from `i18n.js`.

## How to use

```js
import { t } from './i18n.js';

// In JS code:
showToast(t('notes.saved'));
showConfirm(t('notes.deleteConfirm'), [
    { label: t('common.delete'), ... },
    { label: t('common.cancel'), ... }
]);

// With interpolation:
t('notes.selectSession', { n: sessions.length })  // → "전송할 Claude 세션 선택 (3개 실행 중)"
```

## In HTML (index.html):
```html
<span data-i18n="common.save">Save</span>
<input data-i18n-placeholder="notes.titlePlaceholder" placeholder="Untitled">
<button data-i18n-title="preview.back" title="Back">◀</button>
```

## Rules
1. **NEVER hardcode Korean or English text** in JS files. Always use `t('key')`.
2. When adding new text, add keys to BOTH `public/locales/ko.json` AND `public/locales/en.json`.
3. Use dot-separated namespace: `{module}.{key}` (e.g., `notes.saved`, `launch.addItem`).
4. Use `common.*` for shared terms (cancel, delete, save, confirm, close, etc.).
5. Use `{var}` syntax for interpolation: `t('files.selected', { n: count })`.
6. HTML elements use `data-i18n` (textContent), `data-i18n-placeholder`, `data-i18n-title`.
7. After adding `data-i18n` attributes, call `translateDOM()` to apply.

## Locale files
- `public/locales/ko.json` — Korean
- `public/locales/en.json` — English
- Settings key: `general.language` (default: `'en'`)
